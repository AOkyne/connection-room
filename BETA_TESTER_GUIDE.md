## Best Practices for Beta Testing

### Do This
✅ Test on **your actual phone** (not just a web browser)  
✅ Try the **main flows** first (signup → onboarding → explore)  
✅ Test on **WiFi and cellular data** if possible  
✅ **Restart the app** if something seems broken  
✅ **Hard refresh** in browser (hold refresh button)  
✅ Report **specific steps** that caused the issue  
✅ Include **screenshots** if the issue is visual  
✅ Note **what device/OS version** you're using  

### Don't Do This
❌ Don't assume something's broken after one try (try twice)  
❌ Don't report "it's slow" without context  
❌ Don't forget the steps to reproduce the issue  
❌ Don't report the same bug multiple times  
❌ Don't test with made-up data (use realistic names/info)

## Known Limitations (These Are OK)

- **Demo mode**: Some features use demo data that resets on refresh
- **Real-time features**: Updates may take a second or two
- **Email**: Auth emails go to spam sometimes (check spam folder)
- **Mobile**: Some pages optimized for portrait, portrait orientation
- **Large images**: Uploading very large photos may be slow
