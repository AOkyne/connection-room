## Deployment Checklist

### Step 1: Database Schema

Apply the new schema to your Supabase instance:

1. Open Supabase dashboard
2. Go to SQL Editor
3. Copy contents of `lib/supabase/schema-v1.3.sql`
4. Paste and run

Or via CLI:
```bash
supabase db push
```

This creates:
- `daily_companion_content` table
- `weekly_notes` table
- `user_reflections` table
- Necessary indexes and RLS policies

### Step 2: Seed Initial Content

**Option A: Automatic (Recommended)**

Call during app initialization:

```typescript
// In app/app/layout.tsx
import { initializeDailyCompanion } from "@/lib/seed/init-daily-companion";

useEffect(() => {
  initializeDailyCompanion(); // Runs once, seeds if needed
}, []);
```

**Option B: Manual**

Create a one-time script to seed:

```typescript
import { seedDailyCompanionContent } from "@/lib/data/daily-companion";

async function runOnce() {
  const success = await seedDailyCompanionContent();
  if (success) console.log("Seeding complete");
}

runOnce();
```

**Option C: Supabase Function**

Create an Edge Function to seed on first request (if using serverless).

### Step 3: Update Main Dashboard

Replace or enhance the main app dashboard (`app/app/page.tsx`) to use DailyCompanionDashboard:

```typescript
import { DailyCompanionDashboard } from "@/components/daily-companion/DailyCompanionDashboard";
import { ContinueWhereYouLeftOff } from "@/components/daily-companion/ContinueWhereYouLeftOff";
import { FeaturedSpacesCard } from "@/components/daily-companion/FeaturedSpacesCard";

export default function AppHome() {
  return (
    <div className="space-y-8">
      <DailyCompanionDashboard displayName={profile.displayName} userId={userId} />
      
      <div className="border-t border-[#e8ddd2] pt-8">
        <ContinueWhereYouLeftOff profile={profile} recentReflections={reflections} upcomingEvents={events} />
      </div>
      
      <div className="border-t border-[#e8ddd2] pt-8">
        <FeaturedSpacesCard spaces={spaces} />
      </div>
      
      {/* Existing dashboard content below (less prominent) */}
    </div>
  );
}
```

### Step 4: Test

**Demo Mode Testing (No Database)**
1. Start dev server: `npm run dev`
2. Visit app in browser
3. You should see:
   - Greeting
   - Today's theme
   - Reflection prompt
   - All daily content
   - No database errors

**Supabase Testing**
1. Create test user account
2. Sign in
3. Verify daily content loads
4. Save a reflection
5. Check it appears in "My Reflections"
6. Edit the reflection
7. Delete the reflection
8. Come back tomorrow, content should change

### Step 5: Deploy

Push to your hosting:

```bash
# Vercel
vercel deploy

# Or your hosting provider
git push origin main
```

## Breaking Changes

**None!** This is a non-breaking addition. Existing features continue to work.

The only change is the main dashboard now prioritizes daily companion content. You can customize the layout in `app/app/page.tsx`.

## Configuration

### Launch Date

Daily content rotates based on app launch date (currently Jan 1, 2024):

```typescript
// In lib/data/daily-companion.ts
const launchDate = new Date("2024-01-01");
```

To change:
1. Update the date
2. Content rotation will adjust automatically
3. No database migration needed

### Featured Spaces

Customize which spaces appear in FeaturedSpacesCard:

```typescript
// In components/daily-companion/FeaturedSpacesCard.tsx
const FEATURED_SPACE_IDS = [
  "commons",
  "touch-affection",
  "couples",
  "spirituality-sexuality",
  "dating-desire",
];
```

Change to your preferred 5 spaces.

### Content Voice

All content is written in Trevor James' voice. To maintain consistency when adding more content:

**Tone Guidelines:**
- Warm and grounded
- Compassionate but not saccharine
- Masculine but soft
- Mature and thoughtful
- Slightly poetic but clear and direct
- Never cheesy or guru-like
- Never fear-based or shame-based
- Never overly clinical or detached

See `docs/V1.3-DAILY-COMPANION.md` for detailed examples.

3. Reflections saved to localStorage
4. No database errors
5. Perfect for testing, development, or fully offline use

No changes needed; system detects Supabase availability and adapts.

## Monitoring & Support

### What to Watch For

- Slow dashboard loads on first render (seeding takes a few seconds)
- localStorage full errors in demo mode (clear periodically)
- Timezone discrepancies if serving to multiple regions

### Common Issues

**Issue: No daily content displays**
- Check schema was applied: `SELECT COUNT(*) FROM daily_companion_content;`
- Check seeding ran: Should have 720 rows
- Check Supabase connection in app

**Issue: Reflections not saving**
- Check RLS policies applied correctly
- Check user is authenticated
- Check network errors in console
- In demo mode, check browser allows localStorage

**Issue: Content doesn't change daily**
- Launch date might be too recent
- Check getDaysSinceLaunch() returns correct value
- Verify rotation_index values are 0-119
