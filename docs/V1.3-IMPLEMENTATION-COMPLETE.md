## Deployment Steps

### Phase 1: Database Setup (5 minutes)

1. **Open Supabase SQL Editor**
   - Go to https://app.supabase.com/project/[your-project]/sql/new
   - OR go to SQL Editor tab

2. **Copy and paste the schema**
   - Copy entire contents of `scripts/deploy-v1.3-schema.sql`
   - Paste into the SQL editor
   - Click "Run" button

3. **Verify tables created**
   ```sql
   SELECT table_name FROM information_schema.tables 
   WHERE table_schema = 'public' AND table_name LIKE 'daily%';
   ```
   You should see:
   - `daily_companion_content`
   - `weekly_notes`
   - `user_reflections`

### Phase 2: Initialize Dashboard (2 minutes)

1. **Backup current dashboard** (optional but recommended)
   ```bash
   cp app/app/page.tsx app/app/page-backup.tsx
   ```

2. **Replace with v1.3 dashboard**
   ```bash
   cp app/app/page-v1.3.tsx app/app/page.tsx
   ```

3. **Verify imports compile**
   - Start dev server: `npm run dev`
   - Check console for errors
   - Visit http://localhost:3000/app

### Phase 3: Seed Content (1 minute)

The system will automatically seed content on first app load via `initializeDailyCompanion()`. But you can manually seed:

**Option A: Automatic (recommended)**
- Just visit the app—it will seed on first render
- Check database: `SELECT COUNT(*) FROM daily_companion_content;`
- Should show 720 rows

**Option B: Manual seed**
```typescript
import { seedDailyCompanionContent } from "@/lib/data/daily-companion";

// Run once
const success = await seedDailyCompanionContent();
console.log(success); // true if success
```

### Phase 4: Test in Demo Mode (5 minutes)

**Without Supabase:**
1. Stop dev server
2. Temporarily disable Supabase in your env (or it will auto-fallback if not available)
3. Start dev server again
4. Visit app—everything should work with seed data + localStorage
5. Try saving a reflection—it should work via localStorage

**With Supabase:**
1. Create test user account
2. Sign in
3. You should see:
   - Daily theme
   - Reflection prompt with save form
   - Embodiment practice
   - Body check-in
   - Quote
   - Weekly note
4. Save a reflection
5. Check it appears in "My Reflections"
6. Edit and delete it

### Phase 5: Deploy to Production (5 minutes)

```bash
# Commit changes
git add app/app/page.tsx
git commit -m "feat: implement v1.3 daily companion experience"

# Push to main (or your deploy branch)
git push origin main

# Your CI/CD will deploy automatically
# (Vercel, Railway, etc.)
```

Supabase changes deploy automatically—no extra steps needed.

## Key Design Decisions Implemented

1. **Rotation-based content**: Daily content determined by `daysSinceLaunch % 120`
   - All members see same content on same date
   - No admin UI needed to "publish"
   - Scales automatically if you extend content

2. **Private-by-default reflections**: All reflections stored with `privacy_setting: 'private'`
   - Users control visibility
   - Safe for members to share vulnerabilities
   - Future feature to share in spaces

3. **Full demo mode support**: Works 100% offline
   - Seed data available locally
   - localStorage for reflections
   - No database errors
   - Perfect for development

4. **Component-based architecture**: 11 reusable components
   - Easy to modify
   - Easy to extend
   - Easy to test
   - Follow existing design patterns

5. **TypeScript throughout**: Full type safety
   - No `any` types
   - Proper interfaces
   - IDE autocompletion

## Performance

**Query Optimization:**

**Component Performance:**

## Security

**RLS Policies:**
- Users can only access their own reflections
- Daily content is read-only public
- Admins need explicit check (add auth to `/admin`)

**Data Privacy:**
- Reflections are completely private
- No analytics on reflection content
- Users can delete anytime
- No auto-sharing

## Troubleshooting

**Dashboard doesn't load**
- Check browser console for errors
- Verify Supabase URL is correct in `.env.local`
- Check if `initializeDailyCompanion()` is called

**No daily content displays**
- Verify schema was applied: `SELECT COUNT(*) FROM daily_companion_content;`
- Should show 720 rows
- If not, run seeding manually

**Reflections don't save**
- Check network tab for failed requests
- Verify RLS policies are applied
- Check Supabase logs for SQL errors
- In demo mode: check browser allows localStorage

**Content doesn't change daily**
- Launch date might be too recent (all days at index 0)
- Check `getDaysSinceLaunch()` returns correct value
- Verify rotation_index values are 0-119

**Admin interface shows no content**
- Verify seeding completed
- Check if you're authenticated (add auth check)
- Reload page
